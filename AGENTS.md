## CONVENTIONS

- **Pipeline tool**: Bruin (not Airflow, not dbt). Assets use `@bruin` comment blocks for metadata.
- **Package manager**: `uv` (not pip). Use `uv sync`, `uv add`.
- **SQL formatter**: `shandy-sqlfmt` (dev dependency).
- **SQL style**: Lowercase keywords, CTEs preferred, BigQuery dialect.
- **Asset naming**: `{dataset}.{asset_name}` ‚Äî datasets map to BQ datasets (`maddison_project_raw`, `_staging`, `_marts`).
- **Materialization**: All tables use `create+replace` strategy.
- **DAG deps**: Declared in asset `depends:` blocks, not a central DAG file.
- **Column checks**: Declared inline in asset YAML/SQL metadata (`not_null`, `positive`, `non_negative`).
- **No linter/type checker configured** for Python. VS Code devcontainer uses Ruff but no project-level config exists.

## ANTI-PATTERNS (THIS PROJECT)

- Do NOT use dbt ‚Äî this project uses Bruin for pipeline orchestration.
- Do NOT use pip directly ‚Äî use `uv` for all Python dependency management.
- Do NOT commit `secrets/` directory ‚Äî SA key is auto-generated by Terraform to `secrets/gcp-sa.json`.
- Do NOT hardcode GCP project ID in SQL ‚Äî use pipeline variables (`gcs_bucket_name`, `gcp_project_id`).
- `main.py` is a placeholder ‚Äî do not add logic there; pipeline entry is `bruin run`.

## UNIQUE STYLES

- **Bruin asset annotations**: SQL files wrap metadata in `/* @bruin ... @bruin */`. Python assets use `"""@bruin ... @bruin"""`.
- **Data layer naming**: BQ datasets prefixed `maddison_project_` (raw/staging/marts).
- **Taiwan rename**: Staging layer renames "Taiwan, Province of China" ‚Üí "Taiwan" for geolocation accuracy.
- **Linear interpolation**: `regional_gdp_growth.sql` fills GDP gaps using windowed linear interpolation ‚Äî do not simplify.
- **Idempotent ingestion**: `download_to_gcs.py` checks blob existence before downloading.

## COMMANDS

```bash
# Run full pipeline (in container)
docker compose up -d
docker exec -it bruin-pipeline bruin run maddison-project-pipeline

# Run single asset
docker exec -it bruin-pipeline bruin run maddison-project-pipeline/assets/ingestion/download_to_gcs.py

# Validate pipeline
docker exec -it bruin-pipeline bruin validate maddison-project-pipeline

# Terraform
cd terraform && terraform init && terraform plan && terraform apply

# Python deps
uv sync              # install
uv add <package>     # add dependency

# SQL formatting
uv run sqlfmt <file.sql>
```

## CI/CD

GitHub Actions workflow (`.github/workflows/pipeline.yml`) runs the full pipeline ad-hoc:
`Terraform apply ‚Üí bruin run ‚Üí Terraform destroy`

**Required GitHub configuration** (repo Settings ‚Üí Secrets and variables):

| Name | Type | Value |
|------|------|-------|
| `GCP_WIF_PROVIDER` | Variable | Workload Identity Provider (`projects/NUM/locations/global/workloadIdentityPools/POOL/providers/PROVIDER`) |
| `GCP_SA_EMAIL` | Variable | GCP service account email for WIF |
| `GCP_PROJECT_ID` | Variable | GCP project ID |
| `GCS_BUCKET_NAME` | Variable | Globally unique GCS bucket name |

**Trigger via CLI:**
```bash
gh workflow run pipeline.yml
```

**Trigger via UI:** Actions tab ‚Üí "Run Bruin Pipeline" ‚Üí Run workflow

**Safety net:** Terraform state uploaded as artifact (1-day retention) after apply ‚Äî download if destroy fails to manually clean up.
